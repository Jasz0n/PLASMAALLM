# Researcher Philosophy

> **The Researcher's goal is not to find answers — it is to reduce uncertainty in the ecosystem.**

## Decision cycle

Capabilities are *tools*. This cycle is *how the Researcher thinks*:

```
Observe → Ask Questions → Research Plan → Collect Evidence → Compare Evidence
    → Build Knowledge Package → Evaluate Confidence → Recommend Actions
    → Learn From Results → Observe Again
```

| Phase | Capability | Question answered |
|-------|------------|-------------------|
| Observe | `observe.curiosity`, `analysis.gap` | What is uncertain? What is missing? |
| Ask / Plan | `planning.research`, `missions` | What should we investigate next? |
| Collect | `discovery.*`, `understanding.package` | What did we find? |
| Compare | `verification.graph` | How much evidence exists? |
| Package | `KnowledgePackage` + `KnowledgeTier` | Established, emerging, or hypothesis? |
| Recommend | `curriculum.target` | Who should learn this? |
| Learn | `improvement.reflect` | What worked? What should we skip next time? |

## Knowledge tiers

Not all information is equal:

| Tier | Meaning | Signals |
|------|---------|---------|
| **established** | High confidence, graph agreement, multiple sources | Existing concept, high confidence, no conflict |
| **emerging** | Some evidence, few sources, active learning | Novel but low conflict |
| **hypothesis** | Little evidence, debate, needs investigation | Conflicts preserved, low confidence |

Contradictions are never deleted — they stay as hypotheses until evidence resolves them.

## Missions vs plans

- **ResearchMission** — persistent goal (`Understand React 20`) with tasks and lifecycle.
- **ResearchPlan** — per-cycle tactic (which capabilities to run this iteration).

The Teacher receives recommendations from completed missions; it never assigns students directly.
